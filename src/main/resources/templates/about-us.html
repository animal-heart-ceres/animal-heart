Test Driven Development

Test-driven development (TDD) is a new part of Codeup's curriculum. It's a way to develop software where you write the test, let it fail and then go write the code to make it work. Though on the surface it may appear that it takes more time, we saw that it wasn't the case. There were three main things that we noticed from TDD. First, it forces you to think through problems from beginning to end. Second, when passing code to your teammates, there's almost instant proof that the application can run before having to go through time consuming manual testing of your site. And third, it allows back-end developers and front-end developers to effectively work independently of each other. What you're seeing are the actual tests run in our application, we didn't find them on YouTube.
